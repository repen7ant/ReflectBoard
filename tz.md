# Техническое задание: ReflectBoard

## Цель приложения

Помочь пользователю вести полный учёт всего, что он реально делает (запланированное + импульсивное), с обязательной рефлексией и точной статистикой по затраченному времени.

---

## 1. Общая концепция и пользовательский путь

### Первый визит (неавторизованный пользователь)

- Приветственная страница.
- Короткое описание.
- Кнопки: «Зарегистрироваться» / «Войти».

### После входа

- Пользователь сразу попадает на главную страницу — Kanban-доску.

### Роли

Только одна — обычный пользователь.

---

## 2. Основные разделы приложения

### 2.1. Главная страница — Kanban-доска

#### Колонки

| Колонка           | Назначение                              |
| ----------------- | --------------------------------------- |
| **Backlog**       | Всё, что нужно сделать когда-нибудь     |
| **Today**         | То, что планирую сделать именно сегодня |
| **In Process**    | Задачи, над которыми я сейчас работаю   |
| **On Reflection** | Завершённые задачи, ещё без рефлексии   |

#### Возможности на доске

- Добавление новой задачи в любую колонку (кнопка «+» в заголовке каждой колонки).
- Drag & Drop между всеми колонками.
- Полный CRUD задачи (редактировать, удалить) прямо из карточки.
- В каждой карточке отображается: заголовок, категория (учёба, работа и т.д.), дедлайн (если есть), короткое описание (первые 2 строки).

**Что отображается на доске:** все записи где `parent_id IS NULL OR is_on_board = true`, и `status != done`.

#### Переход в «On Reflection» = завершение задачи

При перетаскивании карточки в колонку **On Reflection** (или нажатии кнопки «Завершить» в карточке) обязательно открывается модальное окно рефлексии.

**Поля модального окна рефлексии:**

- Что было сделано? _(большое текстовое поле, обязательное)_
- Сколько времени потрачено? _(отдельные поля часы/минуты, обязательное)_
- Короткая рефлексия / что понял / что можно улучшить _(опционально, но рекомендуется)_
- Категория _(выпадающий список или теги)_
- Дата и время завершения _(автоматически текущее, можно изменить)_

После сохранения задача автоматически убирается с доски и попадает на страницу «Сделанное».

---

### 2.2. Страница «Сделанное» (Журнал выполненных задач)

Хронологический список записей с панелью фильтрации и поиска над ним.

#### Фильтры и поиск

- **Поиск** — строка полнотекстового поиска по заголовку и тексту рефлексии; результаты обновляются в реальном времени (debounce ~300 мс).
- **Фильтр по дате** — быстрые пресеты: «Сегодня», «7 дней», «30 дней», «Всё время»; также произвольный диапазон через date picker. По умолчанию — последние 30 дней.
- **Фильтр по категории** — мультивыбор из списка всех категорий пользователя; при выборе нескольких показываются записи с любой из них.

Все три фильтра работают одновременно (логика AND между группами фильтров). Активные фильтры отображаются в виде dismissible-бейджей под панелью; кнопка «Сбросить всё» снимает их разом.

#### Каждая запись показывает

- Заголовок
- Категория
- Дата и точное время завершения
- Сколько времени потрачено
- Короткий превью рефлексии (первые 2 строки)

#### Действия с записью

Клик по строке → открывается полная карточка с:

- Полным описанием задачи
- Полной рефлексией
- Временем
- Возможностью отредактировать описание, рефлексию или время
- Кнопкой «Удалить»

---

### 2.3. Кнопка «Быстрая запись» (FAB)

Большая плавающая кнопка в правом нижнем углу на всех страницах.

**Назначение:** фиксировать импульсивные действия (например, залипание в YouTube) без создания задачи на доске заранее.

**Поля модального окна:**

- Что я сделал? _(заголовок + описание)_
- Сколько времени потрачено? _(по ощущениям)_
- Категория
- Рефлексия _(короткая)_

После сохранения запись сразу попадает на страницу «Сделанное» как завершённая задача.

---

### 2.4. Проекты и подзадачи

#### Концепция

Проект — это группирующая сущность для нескольких связанных задач. Он не имеет собственного жизненного цикла, прогресс-баров и процентов готовности — это «папка» с подзадачами и суммарным временем по ним. Проект не является отдельной сущностью в БД: это та же запись `activities` с флагом `is_project = true`.

#### Создание проекта

В модальном окне создания задачи есть переключатель **«Это проект»**. При его включении карточка становится проектом — визуально отличается на доске: иконка папки и счётчик подзадач (например, `2/5 выполнено`).

#### Добавление подзадач

Открыть карточку проекта → секция «Подзадачи» со своим «+». Каждая подзадача — это обычная задача, привязанная к проекту через `parent_id`. По умолчанию подзадачи **не отображаются на основной доске** — они живут внутри карточки проекта в виде списка со статусом рядом.

#### Вывод подзадачи на доску

В карточке проекта рядом с каждой подзадачей есть кнопка **«Взять в работу»**. После нажатия подзадача появляется на доске как обычная карточка с бейджем проекта (флаг `is_on_board = true`). После завершения (переход в `On Reflection` → рефлексия → `done`) подзадача уходит с доски и возвращается внутрь проекта со статусом «выполнена».

**Типичный флоу:**

1. Создал проект «Диплом» в `Backlog`
2. Открыл карточку → добавил подзадачи: «Введение», «Глава 1», «Найти источники»
3. Нажал «Взять в работу» на «Введение» → она появилась в `Today`
4. Подвинул в `In Process`, поработал, завершил с рефлексией и временем
5. Карточка ушла с доски; внутри проекта «Введение» теперь отмечена как выполнена, время добавилось к итогу проекта

#### Рефлексия по подзадачам

При завершении подзадачи открывается стандартное модальное окно рефлексии — точно такое же, как для обычных задач. Время каждой подзадачи суммируется и отображается на карточке проекта.

#### Завершение проекта

Проект завершается вручную — кнопка «Завершить проект» внутри карточки. При этом открывается модальное окно итоговой рефлексии по всему проекту; суммарное время проставляется автоматически. После сохранения проект уходит на страницу «Сделанное».

### 2.5. Страница «Статистика» (Analytics Dashboard)

#### Техническая особенность

Данный раздел реализуется как **независимый сервис на PHP**, работающий напрямую с Redis для обеспечения мгновенного отклика и минимальной нагрузки на основную БД.

#### Сценарии использования

- **Оперативный контроль:** просмотр текущего прогресса за день без нагрузки на PostgreSQL.
- **Виджет «Пульс дня»:** наглядное распределение времени по категориям, обновляемое сразу после нажатия «Быстрой записи» или завершения задачи.

#### Визуализации

**Live-данные (из Redis):**

- Live-счётчик: общее количество продуктивных минут за сегодня _(O(1), без запросов в БД)_
- Сравнение с целью: прогресс-бары по ключевым категориям _(например, «Учёба: 120/180 мин»)_
- Динамика активности: распределение импульсивных и запланированных действий за последние 24 часа

**Исторические данные (из PostgreSQL):**

- Круговая диаграмма — распределение времени по категориям за выбранный период (неделя / месяц / всё время)
- Столбчатая диаграмма — сколько часов в день по категориям

**Фильтры:** день / неделя / месяц / произвольный период.

Все расчёты опираются на поле `time_spent_minutes` (единое для всех типов записей).

## 3. Схема базы данных

### Таблица `users` — пользователи

| Поле            | Тип                    | Описание                  |
| --------------- | ---------------------- | ------------------------- |
| `id`            | UUID PK                | Первичный ключ            |
| `email`         | string UNIQUE NOT NULL | Email пользователя        |
| `password_hash` | string NOT NULL        | Хэш пароля                |
| `created_at`    | timestamp              | Дата регистрации          |
| `updated_at`    | timestamp              | Дата последнего изменения |

---

### Таблица `categories` — категории активностей

Категории принадлежат конкретному пользователю — у каждого свой список (учёба, работа, спорт и т.д.).

| Поле         | Тип                | Описание           |
| ------------ | ------------------ | ------------------ |
| `id`         | UUID PK            | Первичный ключ     |
| `user_id`    | UUID FK → users.id | Владелец категории |
| `name`       | string NOT NULL    | Название категории |
| `color`      | string NOT NULL    | HEX-цвет бейджа    |
| `created_at` | timestamp          | Дата создания      |

---

### Таблица `activities` — задачи, проекты, подзадачи, быстрые записи

Единая таблица для всех типов записей. Тип определяется комбинацией флагов `is_project`, `is_on_board`, `is_quick_capture` и наличием `parent_id`.

| Поле                 | Тип                               | Описание                                                  |
| -------------------- | --------------------------------- | --------------------------------------------------------- |
| `id`                 | UUID PK                           | Первичный ключ                                            |
| `user_id`            | UUID FK → users.id NOT NULL       | Владелец записи                                           |
| `parent_id`          | UUID FK → activities.id, nullable | Ссылка на родительский проект (только у подзадач)         |
| `category_id`        | UUID FK → categories.id, nullable | Категория                                                 |
| `title`              | string NOT NULL                   | Заголовок                                                 |
| `description`        | text, nullable                    | Полное описание                                           |
| `reflection_text`    | text, nullable                    | Текст рефлексии                                           |
| `time_spent_minutes` | integer, nullable                 | Потраченное время (заполняется при завершении)            |
| `status`             | enum NOT NULL                     | `backlog \| today \| in_process \| on_reflection \| done` |
| `is_project`         | boolean, default false            | Карточка является проектом                                |
| `is_on_board`        | boolean, default false            | Подзадача выведена на доску («Взять в работу»)            |
| `is_quick_capture`   | boolean, default false            | Быстрая запись через FAB                                  |
| `deadline`           | timestamp, nullable               | Дедлайн                                                   |
| `tags`               | text[], default '{}'              | Массив тегов                                              |
| `completed_at`       | timestamp, nullable               | Дата и время завершения                                   |
| `created_at`         | timestamp                         | Дата создания                                             |
| `updated_at`         | timestamp                         | Дата последнего изменения                                 |

#### Логика флагов

| Тип записи                       | `is_project` | `parent_id`      | `is_on_board` | `is_quick_capture` |
| -------------------------------- | ------------ | ---------------- | ------------- | ------------------ |
| Обычная задача на доске          | false        | null             | false         | false              |
| Проект                           | true         | null             | false         | false              |
| Подзадача (живёт внутри проекта) | false        | `<uuid проекта>` | false         | false              |
| Подзадача, выведенная на доску   | false        | `<uuid проекта>` | true          | false              |
| Быстрая запись (FAB)             | false        | null             | false         | true               |

---

### Связи между таблицами

```
users
 │
 ├──────< categories         (categories.user_id → users.id)
 │
 └──────< activities         (activities.user_id → users.id)
              │
              ├── category_id ──────► categories.id    (nullable)
              │
              └── parent_id ─────────► activities.id   (self-reference, nullable)
                                        подзадачи ссылаются на родительский проект
```

---

### Механизм синхронизации (Dual-Write Strategy)

При завершении любой записи (статус → `done`) или создании быстрой записи (`is_quick_capture = true`) FastAPI выполняет два действия одновременно:

1. **Персистентность:** запись полной копии данных в PostgreSQL.
2. **Агрегация:** инкремент счётчиков в Redis по ключам вида `stats:user:{id}:daily:{date}:category:{category_id}`.

Это обеспечивает мгновенное обновление Live-дашборда без дополнительных запросов в основную БД.

## 4. Дополнительные требования

- **Авторизация:** регистрация / вход (email + пароль, опционально Google OAuth).
- **Мобильная адаптивность:** приложение должно отлично работать с телефона; Kanban-доска удобна в мобильной версии.
- **Быстродействие:** drag & drop без лагов, мгновенное сохранение.
- **Уведомления:** Telegram-бот (напоминание про дедлайны, отправка статистики за день).

## 5. Технический стек

| Компонент                   | Технология                | Назначение                                                                                        |
| --------------------------- | ------------------------- | ------------------------------------------------------------------------------------------------- |
| **Backend (Core API)**      | FastAPI (Python)          | Управление состоянием задач, авторизация, бизнес-логика, Dual-Write в PostgreSQL + Redis          |
| **Backend (Analytics)**     | PHP 8.x (PHP-FPM)         | Сервис генерации аналитических отчётов и Live-дашбордов; читает напрямую из Redis                 |
| **Frontend**                | React + Vite + JavaScript | Основной интерфейс                                                                                |
| **Frontend (Analytics UI)** | PHP-шаблоны               | Страница статистики, встраиваемая через iframe или отдельный роут                                 |
| **Kanban DnD**              | @dnd-kit                  | Drag & Drop на Kanban-доске                                                                       |
| **База данных**             | PostgreSQL 16             | Основное хранилище всех данных                                                                    |
| **Кэш / Агрегации**         | Redis 7.x                 | «Горячее» хранилище агрегированных данных для Live-аналитики; Message Broker для будущих очередей |
| **Инфраструктура**          | Docker Compose            | Контейнеры для каждого компонента                                                                 |

```
    │                         └──► Redis ◄── PHP-FPM (Analytics Service)
    │                                                │
    └── Analytics page ◄──────────────────────────────
```
