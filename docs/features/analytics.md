# Страница «Статистика»

Роут `/analytics` в Laravel. Исторические данные из MySQL через FastAPI, live-данные из Redis.

## Периоды

Скользящие: 7 дней / 30 дней / 90 дней / всё время. Нет деления «за день» — если сессия пересекает полночь, она не разрывается.

## Структура страницы

### Блок 1 — Обзорные метрики (4 карточки)

| Метрика | Описание |
| --- | --- |
| Выполнено задач | Кол-во записей со статусом `done` за период |
| Суммарное время | Сумма `time_spent_minutes` за период |
| Streak | Дней подряд с хотя бы одной выполненной задачей |
| Completion rate | Выполнено / создано за период, в % |

### Блок 2 — Тепловая карта активности (GitHub-style)

Активность по дням на основе `completed_at`. Не `created_at` — день считается активным только если что-то завершено.

### Блок 3 — Категории и теги (два столбца)

**Слева — горизонтальные бары по категориям:**
- Суммарное время (`time_spent_minutes`) и кол-во задач на категорию
- Fallback на `category_snapshot_name` если категория удалена

**Справа — облако тегов:**
- Размер тега пропорционален частоте
- Минимальный порог — тег встречается хотя бы 3 раза за период

### Блок 4 — Live за последние 24 часа (из Redis)

Ключи: `stats:user:{id}:daily:{date}:category:{category_id}`

- Продуктивных минут за последние 24 часа
- Распределение по категориям (прогресс-бары)

> **Важно:** Live-блок показывает данные из Redis которые пишутся через Dual-Write при завершении задачи или создании quick capture. Quick capture — это способ создания записи, не отдельный тип активности. Через FAB можно зафиксировать как спонтанную полезную задачу, так и потраченное впустую время — категории и теги дают пользователю контроль над смыслом.

## Empty states

Если данных за период нет — блоки скрываются и показывается подсказка: «Complete your first task to see stats here».
