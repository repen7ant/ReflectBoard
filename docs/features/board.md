# Главная — Kanban-доска

## Колонки

| Колонка       | Назначение                                |
| ------------- | ----------------------------------------- |
| Backlog    | Всё, что нужно сделать когда-нибудь       |
| Today      | То, что планирую сделать сегодня          |
| In Process | Задачи, над которыми работаю прямо сейчас |

**Условие выборки:** все записи где `parent_id IS NULL OR is_on_board = true` и `status != done`.

## Карточка задачи отображает

- Заголовок
- Категория (цветная точка + название)
- Теги (`#tag`)
- Дедлайн (если есть; ≤3 дня — жёлтый, сегодня — янтарный, просрочено — красный)
- Кнопка завершения (circle-кнопка справа вверху)
- Кнопка `+` (лог времени, только для не-проектов) — правее circle-кнопки, ниже
- Бейдж накопленного времени (правый нижний угол, если время > 0)
- Прогресс-бар подзадач (только для проектов)

## Real-time обновление (WebSocket)

1. Браузер → REST `PATCH /activities/{id}` → FastAPI
2. FastAPI обновляет MySQL → публикует событие в Redis канал `board:{user_id}`
3. FastAPI пушит событие всем WebSocket-соединениям пользователя
4. Все вкладки мгновенно отражают изменение

## Логирование времени (`+` кнопка)

Позволяет фиксировать время до завершения задачи — по кусочкам, в несколько сессий.

- Нажать `+` → ввести минуты → `time_spent_minutes` сразу увеличивается
- Redis-счётчик обновляется мгновенно (Dual-Write), live-аналитика отражает время ещё до завершения задачи
- `time_logged_minutes` отслеживает сколько времени уже попало в Redis; при завершении в Redis идёт только дельта `time_spent - time_logged`

## Модальное окно завершения

Для **обычных задач:**
- Сколько времени потрачено? _(необязательное; предзаполняется накопленным)_
- Рефлексия / что понял / что улучшить _(необязательное)_

Для **проектов:**
- Только рефлексия (время суммируется из подзадач автоматически)
- Завершение заблокировано если подзадач нет или не все выполнены

После сохранения задача уходит с доски на страницу «Сделанное».

## Проекты и подзадачи

Проект — это та же запись `activities` с флагом `is_project = true`. Не отдельная сущность в БД.

**Типичный флоу:**

1. Создать проект в Backlog (переключатель «Это проект» в модалке)
2. Открыть карточку → добавить подзадачи
3. Нажать «Взять в работу» → подзадача появляется на доске (`is_on_board = true`)
4. Завершить подзадачу с рефлексией → уходит с доски, остаётся внутри проекта
5. Завершить проект вручную → итоговая рефлексия, суммарное время проставляется автоматически

## Быстрая запись (FAB)

Плавающая кнопка `+` в правом нижнем углу на всех страницах. Фиксирует сделанное или потраченное время без создания задачи заранее.

**По умолчанию — непродуктивная запись** (`is_productive = false`). Переключатель «Продуктивно» меняет режим.

**Placeholder рефлексии** зависит от режима:
- Продуктивно: «What do you think about it?»
- Непродуктивно: «Worth it? Needed, or mindless?»

**Поля:** Заголовок + описание, Затраченное время, Категория, Теги, Рефлексия.
После сохранения запись сразу попадает на страницу «Сделанное», Redis-счётчики обновляются (Dual-Write).
